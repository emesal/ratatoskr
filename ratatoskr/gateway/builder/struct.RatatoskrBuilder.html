Set the retry configuration for transient error handling.

Controls exponential backoff behaviour when providers return transient errors (rate limits, server errors, network failures). Default: 3 attempts, 500ms initial delay, 30s max delay, jitter on.

Source

pub fn response_cache(self, config: CacheConfig) -> Self

Enable the response cache for deterministic operations.

Caches embedding and NLI responses using an in-memory LRU + TTL cache (moka). Without this call, no cache is allocated and all requests go directly to providers (zero overhead).

ⓘ
Ratatoskr::builder()
    .openrouter(Some("key"))
    .response_cache(CacheConfig::new().max_entries(10_000).ttl(Duration::from_secs(3600)))
    .build()?;

pub fn discovery(self, config: DiscoveryConfig) -> Self

Override the runtime parameter discovery configuration.

Discovery is on by default; this method allows tuning TTL, capacity, etc.

Source

pub fn disable_parameter_discovery(self) -> Self

Disable runtime parameter discovery entirely.

When disabled, parameter rejections at runtime are not cached and subsequent requests may repeat the same unsupported parameter.

Source

pub fn disable_registry_refresh(self) -> Self

Disable automatic background registry refresh at startup.

By default, build() spawns a fire-and-forget task that refreshes the cached remote registry so the next startup benefits from fresh data. Disable this for offline/hermetic environments or testing.

Source

pub fn remote_registry(self, config: RemoteRegistryConfig) -> Self

Set the remote registry configuration.

At build time, the builder loads from the local cache file only (no network I/O). Use remote::update_registry() or rat update-registry to fetch from the network.

Source

pub fn registry_url(self, url: impl Into<String>) -> Self

pub fn routing(self, config: RoutingConfig) -> Self

Set preferred provider routing.

Reorders the fallback chain so the named provider is tried first for each capability. Unset capabilities keep the default order.

ⓘ
Ratatoskr::builder()
    .openrouter(Some("key"))
    .anthropic("key")
    .routing(RoutingConfig::new().chat("anthropic").embed("local"))
    .build()?;

pub fn validation_policy(self, policy: ParameterValidationPolicy) -> Self

Set the parameter validation policy.

Controls how the registry handles requests containing parameters not declared as supported by the provider:

  • ParameterValidationPolicy::Warn — log a warning, proceed with request (default)
  • ParameterValidationPolicy::Error — return UnsupportedParameter error
  • ParameterValidationPolicy::Ignore — silently proceed
